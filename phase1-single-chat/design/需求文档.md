# Phase 1: 单聊基础 - 需求文档

## 一句话目标

两个人能互相发送文字消息，实时收到，离线后登录能看到历史记录。

---

## 功能需求

### 1. 用户系统
- 用户注册（用户名 + 密码）
- 用户登录
- 在线状态显示（在线/离线）

### 2. 好友系统（简化版）
- 通过用户名添加好友
- 好友列表

### 3. 单聊功能
- 给好友发文字消息
- 消息实时推送给对方
- 显示消息发送时间
- 区分自己发的和对方发的

### 4. 历史消息
- 登录后加载最近 50 条消息
- 下拉加载更早消息（分页）

---

## 非功能需求

| 指标 | 目标 | 说明 |
|-----|------|------|
| 延迟 | < 500ms | 消息从发送到对方收到 |
| 并发 | 100 人在线 | 第一阶段目标 |
| 可靠性 | 不丢消息 | 网络抖动时重试 |
| 存储 | 永久保存 | 消息不删除 |

---

## 技术挑战（预计会踩的坑）

1. **WebSocket 连接管理** —— 断线重连、心跳检测
2. **消息顺序** —— 如何保证不乱序
3. **消息可靠性** —— 如何确保不丢失
4. **数据库设计** —— 消息表怎么设计支持分页

---

## 验收标准

- [ ] 用户 A 注册并登录
- [ ] 用户 B 注册并登录
- [ ] A 添加 B 为好友
- [ ] A 给 B 发消息，B 实时收到
- [ ] B 回复，A 实时收到
- [ ] B 退出登录，A 继续发消息
- [ ] B 重新登录，看到离线期间的消息
- [ ] 能加载历史消息
