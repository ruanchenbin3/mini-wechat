# Phase 1: 技术选型文档

## 技术栈确认

| 层级 | 选择 | 理由 |
|-----|------|------|
| **前端** | React + Vite | 现代、快速、生态好 |
| **后端** | Node.js + Express | 事件驱动适合高并发，JavaScript 全栈 |
| **数据库** | MongoDB | 消息数据 schema 灵活，适合快速迭代 |
| **缓存** | Redis | 在线状态、会话管理 |
| **实时通信** | Socket.io | 封装了 WebSocket，自动处理降级和重连 |
| **部署** | Docker | 方便本地开发和后续部署 |

---

## 备选方案对比

### 后端语言

| 选项 | 优点 | 缺点 | 选择 |
|-----|------|------|------|
| Node.js | 事件驱动适合 IM，JS 全栈 | 单线程 CPU 密集型弱 | ✅ 选定 |
| Go | 性能高，并发原生支持 | 需要学新语言 | 备选 |
| Python | 开发快，库多 | GIL 限制并发 | 备选 |

**结论：** 先学 Node.js，后期需要性能优化时再考虑 Go。

### 数据库

| 选项 | 优点 | 缺点 | 选择 |
|-----|------|------|------|
| MongoDB | Schema 灵活，适合消息 | 事务支持弱 | ✅ 选定 |
| MySQL | 成熟，事务强 | Schema 变更麻烦 | 备选 |
| PostgreSQL | 功能全面 | 学习曲线陡 | 备选 |

**结论：** MongoDB 适合快速迭代，后期需要复杂事务时再考虑迁移。

### 实时通信

| 选项 | 优点 | 缺点 | 选择 |
|-----|------|------|------|
| Socket.io | 自动降级、重连、房间管理 | 包体积大 | ✅ 选定 |
| 原生 WebSocket | 轻量、标准 | 需自己处理重连等 | 备选 |

**结论：** Socket.io 省时间，先用它，后期优化时再考虑原生 WS。

## 开发环境

### 需要安装

- Node.js 18+
- MongoDB 6+
- Redis 7+
- Docker（可选）

### 启动命令（规划）

```bash
# 后端
cd phase1-single-chat/code/server
npm install
npm run dev

# 前端
cd phase1-single-chat/code/client
npm install
npm run dev
```

## 风险与应对

| 风险 | 可能性 | 应对 |
|-----|--------|------|
| Socket.io 学不会 | 低 | 文档丰富，社区大，可问 AI |
| MongoDB 性能不够 | 中 | Phase 1 数据量小，后期可优化 |
| 时间超期 | 中 | 优先保证核心功能（发消息），次要功能可裁剪 |
