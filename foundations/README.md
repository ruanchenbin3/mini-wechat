# 计算机系统底层基础（物理人视角）

## 为什么物理学家需要学这个？

物理学研究自然规律，计算机是实现规律的工具。理解计算机如何工作，就像理解实验仪器的工作原理——不是为了造仪器，而是为了更好地使用它，知道它的边界在哪里。

## 核心类比

| 计算机概念 | 物理实验室类比 | 核心问题 |
|-----------|--------------|---------|
| **进程** | 独立的实验室 | 一个程序如何独占资源？ |
| **线程** | 实验室里的多个实验台 | 如何在同一个程序里同时做多件事？ |
| **内存** | 实验室的储物架 | 数据存在哪里？如何快速取用？ |
| **网络** | 邮寄系统 | 两台独立的计算机如何交换信息？ |
| **文件系统** | 图书馆 | 大量数据如何组织、存储、检索？ |

## 学习路径

### 第一阶段：单机系统（第1-2周）

**01-进程与线程**
- 从"程序"到"进程"：静态代码如何变成动态执行？
- 进程的物理组成：代码段、数据段、堆栈
- 为什么需要线程？线程切换的物理代价
- 并发 vs 并行：真正的同时 vs 快速切换

**02-内存模型**
- 内存的物理结构：从寄存器到硬盘
- 为什么需要虚拟内存？地址翻译的物理意义
- 缓存机制：局部性原理（像不像热力学？）
- 进程间通信：共享内存 vs 消息传递

### 第二阶段：网络通信（第2-3周）

**03-网络协议栈**
- 两台计算机如何"对话"：从物理电信号到应用数据
- 分层模型：每一层解决什么问题？
- TCP 的可靠性：如何在不可靠的信道上实现可靠传输？
- Socket 的本质：操作系统提供的通信端点

**04-文件系统**
- 数据的持久化：从内存到磁盘
- 文件、目录、inode：组织数据的结构
- 数据库的本质：文件系统上的高效索引

## 学习方法

1. **先建立物理直觉** —— 每个概念都找实验室/生活中的类比
2. **动手验证** —— 用简单的代码观察系统行为
3. **画图理解** —— 画内存布局、数据流向图
4. **问为什么** —— 每个设计都有物理约束（速度、成本、可靠性）

## 与即时通讯的关联

学完这些，你会理解：
- 为什么 WebSocket 需要"握手"？（TCP 连接的本质）
- 为什么需要 Redis？（内存 vs 磁盘的速度差异）
- 为什么 MongoDB 比文件快？（索引的物理意义）
- Node.js 的"单线程"到底是什么意思？（事件循环的物理实现）
