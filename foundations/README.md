# 计算机系统底层基础

## 核心问题：为什么计算机要发明成现在这样？

这不是一个技术问题，而是一个**物理约束下的工程优化问题**。

计算机的每一个设计，都是为了解决一个具体的物理限制：
- **速度差异** —— CPU 太快，硬盘太慢
- **成本差异** —— 内存贵，硬盘便宜
- **物理距离** —— 电信号传播需要时间
- **能量限制** —— 散热、功耗

理解这些约束，就能理解为什么计算机是现在的结构。

---

## 从第一性原理出发

### 计算机的本质是什么？

**计算机 = 一个能够自动执行指令的物理系统**

就像：
- 蒸汽机 = 将热能转化为机械能的系统
- 电动机 = 将电能转化为机械能的系统
- 计算机 = 将电能转化为信息处理能力的系统

### 为什么需要"程序"？

**物理约束：** 人类思维速度慢，电信号速度快

人类思考一个计算步骤需要秒级，而电路完成同样的计算需要纳秒级（10^-9 秒）。

**解决方案：** 人类一次性写好所有步骤（程序），然后让电路自动高速执行。

这就是**存储程序计算机**的核心思想——程序和数据都存在存储器里，CPU 自动取指令、执行。

### 为什么需要"存储层次"？

**物理约束：** 速度快的存储贵，便宜的存储慢

| 存储类型 | 访问时间 | 成本（每GB）| 物理原因 |
|---------|---------|-----------|---------|
| CPU 寄存器 | 0.3 ns | $10,000+ | 直接在芯片上，电路最短 |
| L1 缓存 | 1 ns | $1,000+ | 靠近 CPU 的 SRAM |
| L2/L3 缓存 | 3-10 ns | $100+ | 芯片上的更大 SRAM |
| 内存 (DRAM) | 100 ns | $10 | 需要刷新电容，离 CPU 较远 |
| SSD | 100 μs | $0.1 | 闪存，电子隧穿需要时间 |
| 机械硬盘 | 10 ms | $0.02 | 物理磁头移动，机械运动 |

**关键洞察：** 速度差 10^7 倍（寄存器 vs 硬盘），成本差 10^6 倍。

**工程权衡：** 用少量快速存储 + 大量慢速存储，通过**局部性原理**优化平均访问时间。

## 学习路径（从物理约束出发）

### 01-速度差异：为什么需要进程和线程？

**核心问题：** CPU 太快，I/O 太慢，如何不浪费 CPU？

**物理约束：**
- CPU 执行指令：纳秒级
- 内存访问：百纳秒级
- 磁盘 I/O：毫秒级
- 网络 I/O：几十到几百毫秒

**工程解决方案：**
- 多道程序 → 进程
- 并发执行 → 线程
- 异步 I/O → 事件驱动

### 02-空间约束：为什么需要虚拟内存？

**核心问题：** 物理内存有限，但程序需要更多地址空间

**物理约束：**
- 内存芯片贵且功耗大
- 程序需要连续的地址空间
- 多个程序需要隔离

**工程解决方案：**
- 虚拟内存 → 地址翻译 → 页表
- 页面置换 → LRU 算法
- 写时复制 → 节省物理内存

### 03-距离约束：为什么需要网络协议栈？

**核心问题：** 两台计算机如何可靠通信？

**物理约束：**
- 电信号传播速度：光速的 2/3（光纤中约 20 万公里/秒）
- 北京到纽约：物理距离 11,000 公里 → 信号传播 55ms
- 信号会衰减、会受干扰
- 网络会拥塞、会丢包

**工程解决方案：**
- 分层设计 → 每层解决特定问题
- TCP → 可靠传输、拥塞控制、流量控制
- 协议栈 → 模块化、可替换

### 04-持久化约束：为什么需要文件系统？

**核心问题：** 断电后数据如何保存？

**物理约束：**
- 内存（DRAM）需要持续供电，断电即失
- 磁盘（磁/闪存）断电保持，但访问慢
- 大量数据需要组织、索引、检索

**工程解决方案：**
- 文件系统 → 组织磁盘空间
- 索引结构 → B树/LSM树 → 快速查找
- 日志结构 → 保证一致性

## 学习方法

1. **先问为什么** —— 这个设计解决了什么物理约束？
2. **再问为什么不** —— 有没有其他方案？为什么被否定了？
3. **动手验证** —— 用简单实验观察系统行为
4. **量化理解** —— 记住关键数字（ns, μs, ms）

---

## 关键数字（必须记住）

```
时间尺度：
1 纳秒 (ns) = 10^-9 秒  ← CPU 执行一条指令
1 微秒 (μs) = 10^-6 秒  ← SSD 访问
1 毫秒 (ms) = 10^-3 秒  ← 硬盘访问、网络延迟
1 秒       = 10^0 秒    ← 人类感知

速度对比：
CPU 寄存器访问：~0.3 ns
内存访问：       ~100 ns    (差 300 倍)
SSD 访问：       ~100 μs    (差 300,000 倍)
硬盘访问：       ~10 ms     (差 30,000,000 倍)
网络（同机房）： ~0.5 ms
网络（跨洋）：   ~150 ms

空间尺度：
L1 缓存：   ~32 KB    （CPU 芯片上）
L2 缓存：   ~256 KB   （CPU 芯片上）
L3 缓存：   ~8-32 MB  （CPU 芯片上）
内存：      ~8-64 GB  （主板上的 DIMM 条）
SSD：       ~256 GB - 数 TB
硬盘：      ~1-20 TB
```

**核心洞察：** 每一层速度差 100-1000 倍，这就是存储层次存在的根本原因。
