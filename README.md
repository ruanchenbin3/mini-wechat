# Mini WeChat

一个极简即时通讯系统，从单聊起步，逐步支持群聊、语音房间等功能。

## 项目目标

用 3-6 个月时间，从零开始实现一个可用的即时通讯系统，过程中掌握：
- 实时通信技术（WebSocket/WebRTC）
- 消息可靠性与顺序保证
- 高并发架构设计
- 音视频处理基础

## 功能路线图

### Phase 1: 单聊基础（第 1-3 周）
- [ ] 用户注册/登录
- [ ] 两人文字聊天
- [ ] 消息实时推送
- [ ] 历史消息查看

### Phase 2: 消息增强（第 4-6 周）
- [ ] 已读回执
- [ ] 离线消息
- [ ] 图片/文件传输
- [ ] 消息撤回

### Phase 3: 群聊功能（第 7-10 周）
- [ ] 创建/加入群聊
- [ ] 群成员管理
- [ ] 群消息广播
- [ ] @某人功能

### Phase 4: 语音房间（第 11-16 周）
- [ ] 创建语音房间
- [ ] 上麦/下麦
- [ ] 多人语音通话
- [ ] 语音转文字（AI 辅助）

### Phase 5: 优化与扩展（持续）
- [ ] 性能优化（支持万人在线）
- [ ] 移动端适配
- [ ] 端到端加密

## 技术栈

| 层级 | 技术 |
|-----|------|
| 前端 | React + WebSocket |
| 后端 | Node.js + Express |
| 数据库 | MongoDB（消息）+ Redis（缓存/在线状态）|
| 实时通信 | Socket.io |
| 音视频 | WebRTC |
| 部署 | Docker |

## 学习原则

1. **先跑通，再优化** —— 第一阶段结束就要能聊天
2. **每个阶段有交付** —— 可演示、可复盘
3. **记录踩坑过程** —— 这是最有价值的部分
4. **用 AI 辅助，不依赖 AI** —— AI 找答案，自己做决策
