# Mini WeChat 
笔者最近对即时通讯技术产生兴趣。

一个潜在的心理是“当AI越来越像人，人反而需要一个确认自身人性的价值锚点”，所以真实真诚的人际链接是不可或缺的，即时通讯是少数几个人类能够保留“不完美”和“私密”权利的地方。

这是一个以学习为目的的即时通讯系统项目。每个功能都有详细的学习文档，每一步都有 Git 记录。

## 项目原则

1. **学习优先于功能** —— 不追求代码完美，追求理解透彻
2. **文档即学习笔记** —— 每个功能都要有详细的学习文档
3. **小步提交** —— 每个小进展都推送到 GitHub
4. **复盘总结** —— 每个阶段结束写技术博客

## 文档规范

每个功能必须包含：
1. **功能说明** —— 要做什么
2. **技术原理** —— 底层机制
3. **代码实现** —— 关键代码 + 注释
4. **踩坑记录** —— 遇到的问题和解决
5. **学习总结** —— 学到了什么

## Git 提交规范

```
类型: 简短描述

详细说明（可选）
```

**类型：**
- `Init` —— 初始化
- `Learn` —— 学习笔记/文档
- `Feat` —— 新功能
- `Fix` —— 修复
- `Refactor` —— 重构
- `Docs` —— 文档更新

**示例：**
```
Learn: WebSocket 握手过程

- 理解了 HTTP Upgrade 机制
- 记录了握手报文格式
- 用 Wireshark 抓包验证
```
